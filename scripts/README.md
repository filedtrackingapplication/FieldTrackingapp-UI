# Test Data Automation Scripts

Scripts to seed and clean up test users and customers in the TrackForce app.

---

## Prerequisites

```bash
pip install -r scripts/requirements.txt
```

The backend must be running and reachable (e.g. via `docker-compose up`).

---

## Quick Start

```bash
# 1. Start the stack
docker-compose up -d

# 2. Seed test data (uses defaults: localhost:8000, password Test@1234, 15 drivers)
python scripts/seed_test_data.py

# 3. Run your tests …

# 4. Clean up (soft deactivate — reversible)
python scripts/cleanup_test_data.py
```

## What Gets Created

| Role         | Count   | Employee ID prefix | Email pattern              |
|--------------|---------|-------------------|----------------------------|
| Admin        | 2       | `TEST-ADM-###`    | `admin##@testenv.example.com`       |
| Manager      | 10      | `TEST-MGR-###`    | `manager##@testenv.example.com`     |
| Field Agent  | 10      | `TEST-AGT-###`    | `agent##@testenv.example.com`       |
| Driver       | 10–20   | `TEST-DRV-###`    | `driver##@testenv.example.com`      |
| **Customers**| **40**  | —                 | `customer##@testenv.example.com`    |

All test entities are identifiable by the `[TEST]` prefix in their `full_name`.

## Seed Script

```
python scripts/seed_test_data.py [options]

Options:
  --base-url URL       Backend URL           (default: http://localhost:8000)
  --password SECRET    Password for all accounts (default: Test@1234)
  --drivers N          Number of drivers, 10–20 (default: 15)
  --dry-run            Print plan without calling the API
```

A `scripts/test_manifest.json` file is written on success. It contains every
created entity's ID and is required by the cleanup script.

---

## Cleanup Script

```
python scripts/cleanup_test_data.py [options]

Options:
  --base-url URL   Backend URL (default taken from manifest)
  --hard-delete    Permanently purge records from DB (irreversible)
```

### Soft delete (default)

- Users  → `is_active = false` via `DELETE /api/agents/{id}`
- Customers → `is_active = false` via `PUT /api/customers/{id}`

Records stay in the database but are excluded from all normal API queries.

### Hard delete (`--hard-delete`)

Permanently removes rows using SQLAlchemy directly. Cascades delete all
dependent records (visits, orders, location logs, expenses, odometer logs,
punch records, inventory assignments).

**Requirements for hard delete:**
- Run from the backend Python environment (same virtualenv / container).
- Or inside the backend container: `docker exec -it trackforce_backend python /path/to/cleanup_test_data.py --hard-delete`

---

## Manifest file

`scripts/test_manifest.json` is auto-generated by the seed script and auto-deleted
by the cleanup script. Do **not** commit it to version control.

---

## Notes

- All test accounts share the same password (configurable via `--password`).
- Managers are evenly distributed across 10 zones.
- Field agents and drivers are assigned to managers in round-robin fashion.
- Customers are distributed round-robin across field agents.
- Re-running seed while a manifest exists will fail; run cleanup first.
